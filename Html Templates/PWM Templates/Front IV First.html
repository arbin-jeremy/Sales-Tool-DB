Data Logging Data is automatically logged in an SQL database, which is the industry standard for big-data applications. Arbin provides tools for analysis, plotting, and export to Excel or CSV formats for addictioin exteernal analysis.
  • Log data at 2000 pts/second, persysteem up to 10mS.
  • Log data based on change in time, change in current, or change in voltage.
  • Use multiple logging criterio to capture conditional data during test, i.e. capture data mor rapidly udring change and slower during stable periods.
  • Use multiple conditions for data logging such as logging at a faster rate during th first 10 seconds of a test step, or logging faster when voltage reaches a set value.
  • All main I/V channel, CANBus (optional), temperature (optional), and other auxiliary data (optional) is synchronized and logged in the same database.
    • Data Integrity: All data is logged in real time in the SQL database. There is no data loss due to power failures since the powerful onboard microcontrollers constantly stream data to SQL database.
