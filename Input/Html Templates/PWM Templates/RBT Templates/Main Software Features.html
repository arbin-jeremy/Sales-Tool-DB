Arbin Main Software Features
Arbin’s MITS8 software uses logical menu-driven programming to set control types, termination conditions, and data logging conditions for tests. All results data is automatically saved in an SQL database. The MITS8 software provides tremendous control to customize tests with over 90 meta variables in addition to using numeric values, and users can save up to 16 custom meta-variable during the test to utilize later in the test. Each step can have multiple modes of data logging, and a wide array of auxiliary options is available to expand the interface and control capabilities.
Features Control Type Description
Basic Controls • Current Output constant current.
• Voltage Output constant voltage.
• C-Rate Output value is based on device capacity as entered in batch file.
• Rest Open circuit voltage measurement.
• Power Output constant power.
• Load Output constant resistance.
• Set Variables(s) Perform loops, cycles, reset capacity values, etc.
• Current Ramp Generate a current ramp with specified scan rate.
• Voltage Ramp Generate a voltage ramp with specified scan rate.
• Current Staircase Generate a current staircase with specified magnitude.
• Internal Resistance Apply 10 pulse trains that will indicate the IR trend.
• CCCV Combines constant-current and constant-voltage in one single step.
Simulation Control • Current Simulation Time-vs-Current data may be input directly from a text file.
• Power Simulation Time-vs-Power data may be input directly from a text file.
User-Defined Meta-Variables Multiple custom meta-variables are available that can be assigned before or during a test. This allows a measurement/datapoint to be saved and used later in the test.
Formula The Formula option enables the user to control and limit schedule steps according to dynamic mathematical equations using variables rather than fixed control value
One-to-Many Virtual Mapping Add flexibility of mapping auxiliary channels with main I/V channels. Use any number of aux inputs with each I/V channel. Configurations can be 1:1, 1:many, many:1, or anything in between.
Parallel Channels Ability to group channels together to increase the current capability. Channels must be securely attached to the device being tested. An improper or loose connection may damage the system.
Barcode Scan or manually enter barcode of device under test for each channel when starting tests. Barcode is saved with results data and is searchable when accessing results data.
Auto-Tuning Arbin systems use novel auto-tuning digital voltage control. The Arbin system will automatically optimize its response time based on the IR of device under test. Users have the ability to manually adjust also.
Safety & Protections
System includes a dedicated microcontroller for each channel module, and a redundant microcontroller that is dedicated to monitoring safety limits
System provides global safety limits for each test, and each step can have its own specific safety limits.
Assign user-defined “test object” battery profiles that automatically populates most safety limits in a test.
Intelligent behavioral safety checks in addition to traditional measured limits such as if charge/discharge happens too fast, capacity exceeds an expected value, unexpected change in voltage, etc.
Use mathematical expressions and logical AND/OR conditions to combine multiple safety or control limits.
Create Conditional Safety Limits such entering a “cool down” step if temperature reaches a limit, then continuing the test; but stopping the test if temperature reaches the limit a second time within a defined period of time.

Data Integrity: All data is logged in real time in the SQL database. There is no data loss due to power failures since the powerful onboard microcontrollers constantly stream data to SQL database.
